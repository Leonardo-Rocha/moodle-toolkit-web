Exemplos

Ao realizar uma conversão, o PDF inserido é retornado como um arquivo com a extensão .md. Para que seja possível gerar os arquivos de saída em TeX ou GIFT é necessário "marcar" o arquivo md seguindo alguns padrões estabelecidos.

Abaixo há uma série de exemplos mostrando como marcar cada elemento de texto.

Questão

Dada uma questão, marque seguindo o formato #Q(numero-da-questao).

QUESTÃO 46
Em um modelo de dados que descreve a publicação acadêmica de pesquisadores de diferentes instituições em eventos acadêmicos, considere as tabelas abaixo.

#Q46
Em um modelo de dados que descreve a publicação acadêmica de pesquisadores de diferentes instituições em eventos acadêmicos, considere as tabelas abaixo.

Alternativas

Para cada alternativa, substitua por -(letra)., sendo que a letra deve ser minúscula e a primeira alternativa deve ser a -a..

A lógica nebulosa.
B árvores de decisão.
C redes neurais artificiais.
D ACO (do inglês, Ant-Colony Optimization).
E PSO (do inglês, Particle Swarm Optimization).

-a. lógica nebulosa.
-b. árvores de decisão.
-c. redes neurais artificiais.
-d. ACO (do inglês, Ant-Colony Optimization).
-e. PSO (do inglês, Particle Swarm Optimization).

Figuras

Para cada figura, adicione a marcação ![Texto Alternativo](fig-0000.jpg "legenda-opcional"). Essa marcação deve ser feita em uma única linha e o nome do arquivo deve seguir o padrão fig-(numero).jpg (a única extensão aceita por enquanto é JPG).

O nome do arquivo é incrementado com uma pasta: (nome-do-arquivo-md)_figuras/fig-(numero).jpg. Isso implica que as imagens geradas devem ser salvas nessa pasta.

Em caso de omissão da legenda, as aspas devem ser incluídas sem conteúdo: ![Texto Alternativo](fig-0000.jpg "").

Na conversão em TeX o texto alternativo é ignorado e a legenda é criada como caption. No caso do GIFT o texto alternativo permanece por acessibilidade.

Equações

As equações devem ser escritas diretamente no formato TeX, pois também é renderizado no Markdown, e marcadas com o formato \( equacao_em_TeX \).

\(\exists aluno(x) \wedge \forall y [disciplina(y) \wedge dificil(y)] \wedge gosta(x, y).\)

Código

Código pode ser marcado de duas formas, em uma única linha ou multilinha. No multilinha é possível escolher uma linguagem para fazer o realce de sintaxe. No caso de linha única por simplicidade é omitida a escolha, sendo por padrão SQL.
É recomendado iniciar uma nova linha para abrir código multilinha.

Linha Única

`SELECT * FROM users;`

Multilinha

```sql
SELECT CodDepartamento
FROM EMPREGADO
WHERE AVG (Salario) > 2000
GROUP BY CodDepartamento
```

Quebra de Linha

É possível quebrar linha inserindo linhas vazias ou forçando através do marcador <br/>.
